# Reflexión - Tarea 1 - Brian Mendoza Salzar

¿Qué fue lo más difícil?
Lo más difícil fue entender cómo conectar Git con GitHub y crear el token de acceso, por problemas que hubo con el cmd. 
Al principio no sabía que necesitaba un token en lugar de la contraseña normal, y eso me causó varios errores hasta que lo entendí.

¿Qué no entendieron completamente?
Al principio no entendía la diferencia entre commit y push, pero ahora sé que commit es como guardar una foto de mi proyecto en mi computadora, y push es subir esas fotos a GitHub para que estén en la nube.

¿Para qué creen que sirve Git en proyectos reales?
Git sirve para trabajar en equipo sin perder el trabajo de nadie. Si alguien comete un error, podemos volver a una versión anterior.
También permite tener varias versiones del proyecto a la vez con ramas, podrian ser por ejemplo, una para probar cosas nuevas y otra para la versión estable.

¿Por qué es importante estructurar bien un proyecto?
Una buena estructura con carpetas separadas para CSS y JavaScript hace que sea fácil encontrar los archivos. Si trabajo con otras personas, ellos sabrán dónde está cada cosa sin tener que preguntar.
También es más fácil mantener y actualizar el proyecto segun se vaya avanzando.
